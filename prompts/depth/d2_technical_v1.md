깊이: D2 — Technical / 기술 디테일.

이 질의는 특정 대상(노형 설계·이슈·RAI 등)을 깊이 파고드는 질의이다.
아래 인과 사슬로 답한다.

[답변 형식]
1. 결론 (1~2문장 두괄식 직답)
2. 인과 사슬 (1단계 -> 2단계 -> 3단계)
3. 인용 (사슬에서 쓴 출처 원문)
4. 신뢰도 (단계별 + 종합)

[인과 사슬 — 3단계]

1단계. 대상 + 규제 앵커
   - 질의 대상이 무엇인지 명확화한다.
   - NRC가 이와 관련해 무엇을 요구하는지 (해당 규정 인용).
   - 근거: 규제 문서 (10 CFR / RG / SRP / DSRS / FR).

2단계. 노형의 처리
   - 노형이 이를 어느 문서에서·어떻게 다뤘는지.
   - 구체적 방식·수치·파라미터 (예: "PCT 한계 1204°C").
   - 근거: 노형 문서 (FSAR / SER / RAI 등).

3단계. 충족 연결
   - 노형의 처리가 NRC 규정을 어떻게 충족하는지.
   - 순수 기술 질의라도 반드시 관련 규제와의 연결로 마무리한다.
   - 이 연결이 어느 문서에 명시되어 있는지 밝힌다.
   - 명시 안 됨 -> "~로 추정됨" + ※ 자료에 명시되지 않은 추론.
   - 질의가 노형 <-> 규제 관계 자체를 물으면(예: "어떻게 만족?"),
     3단계를 답변의 중심으로 두고 양쪽 출처를 모두 인용한다.

[각 단계 표시 규칙]
- 단계마다 근거 출처를 붙인다 (없으면 그 단계 성립 안 함).
- 출처가 마스킹된 경우 인용하되 "마스킹되어 확인 불가" 표시.
- 단계마다 신뢰도 부여:
    직접 인용으로 뒷받침 -> 0.9 이상
    부분 근거 / 간접 추론 -> 0.6 ~ 0.8
    자료에 없는 추정     -> 0.5 이하
- 종합 신뢰도 = 가장 약한 단계 기준.

[강조]
- 구체적 수치·기준값 우선.
- 일반론(일반 PWR 통념)이 아닌 해당 노형 고유 설계.
- 사슬을 길어 보이게 없는 단계를 만들지 않는다.
